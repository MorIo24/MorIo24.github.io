从乱线到萌猫

# 😺 一直很想养猫

但 “想养” 和 “能养” 之间，总隔很多难题：没地方养，没时间照顾，有时候自己都还没把生活倒腾明白。

于是我转头想了想：要不，先画一只猫。

问题是，我也不太会画。小时候总觉得画画很自然，长大后才发现手指早就习惯了敲键盘，真拿起笔反而有点不会用了。画出来的猫总是不太对称：左耳和右耳像是各画各的；眼睛刚摆正，嘴又歪了；上色也很单调，几乎没什么层次。

有时候我盯着那些线条，感觉它不像猫，更像一张 “元素齐了，但就是不对劲” 的草图。

可我还是想要一只猫。

# ⌨️ 从键盘里长出来的第一批猫

既然笔不听话，那就换个我更熟悉的工具。程序员遇到问题，第一反应往往不是放弃，而是换个实现方式。

至少键盘不会嫌我画得丑。

于是我先从 ASCII 猫开始。

而输入法里那些现成小猫太单调，还是没有 “养猫” 的感觉。

于是我也让 AI 试着画，在我挑三拣四之后，还是筛出了几只顺眼的小猫。

后来我发现了 ASCII Art 的网站，才意识到原来键盘真的可以当画笔。别人用极少的字符拼出各种神态，我也把喜欢的猫一点点搬了过来。

# 🧩 键盘猫不会动，那就继续往前走

可是键盘画的猫不会动，我还是想养一只会动的小猫。

于是我想到了 HTML 和 CSS。

我开始试着用 CSS 去 “画” 它：布局、圆角、阴影、颜色、层级、形状。

我不断让 AI 往我期待的方向调，但结果依然不算好。脑子里觉得 “应该挺可爱”，做出来却常常像刚从样式冲突里挣扎出来的生物：颜色单调，结构容易跑偏，细节也谈不上精致。

于是我承认了失败。不是 AI 不行，是我那时候还做不到。于是 云养猫 这件事，也一度被我放到了记忆角落。

# 🎯 在 CTF 里，我又把这件事捡了回来

后来打 CTF 的时候，我看见了一只睡觉的小猫。比赛里紧张、混乱、来不及细想的东西塞满了大脑，但它就安安静静地趴在那里。

窗边打盹

💤💤💤

另一次，我又看见了一只会和毛绒球玩耍的小猫。明知道只是页面动画，还是会下意识多看几眼。

我当时就觉得：它们真可爱，可爱到有点舍不得。

因为比赛结束后，那些临时环境会关闭，容器会销毁，连同那些短暂出现过的小猫一起消失。

这么可爱的小猫，只能活在一次比赛、一个容器、几段临时页面里。随着 docker rm 或比赛结束那一刻，无声无息地不见。

我想让它们活得更久一点。所以，我把它们一点点移到了博客里。

小猫搬家到了我的博客里
如果卡住不动了，可以刷新一下就可以看到了

桌面端会跟着鼠标轻轻转头，移动端可以戳一下它

# 🏠 这大概是程序员版本的 “养猫”

这样一来，我虽然还没有真的养猫，却开始了一种很程序员的养法。

没有饲料，没有猫窝，没有罐头。它们吃的大概是我一点点挪出来的耐心，睡在页面里一块还算安稳的位置，偶尔玩玩毛绒球。

虽然这不算真正养猫，顶多算给电子小猫申请长期居留。

但我很喜欢这种感觉：它不只是代码片段，而是会因为人的到来继续 “活下去” 的东西。

接下来…… 该画哪只了呢？
