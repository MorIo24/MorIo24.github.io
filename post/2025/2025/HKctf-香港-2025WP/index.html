# JN

这题的 validate () 先把 flag 固定切成两段：长度必须 22，只取中间 flag [5:21] 的 16 个字符参与校验，再拆成前 8 字符和后 8 字符分别验证。前 8 会走 Java 的 J_Validate，后 8 会走 native 的 N_Valildate，所以整体形态基本就是 flag {(8 字节)(8 字节)}。

Java 这段用的是 RC4-like 流加密，J_Validate 比较的是 “加密结果是否等于常量”，而流加密是可逆的，所以把常量再跑一遍同样的函数就能直接反推出前 8 字符。native 这段是 TEA-like 的 32 轮混淆，最后要求输出等于固定的两组 32 位常量；因为每轮都是 32 位加 / 异或 / 移位组合，可逐轮把加法换成减法、轮常量反向回退，从目标常量倒推回输入，从而得到后 8 字符。

# onebyone

Java 端流程

输入必须 24 位。calculate () 把 24 字符分成 3 组（每组 8 字符→1 个 64bit），每组做 64 轮移位 + 异或变换，得到 3 个 long；再把 3 个 long 按小端拆成 24 个字节组成 arr。

native 的 jiami 是流加密（RC4 变体）

先还原 key：sub_DB0/sub_10D0 互逆，所以 key 就是常量对应的 8 字节："hswSss] e"。

sub_10F0 + sub_1190 是 RC4 的初始化和 KSA，sub_1280 是 PRGA（只是 i 每次 +2 的变体），最终输出是 out [i] = in [i] ^ keystream [i]。

利用流加密可逆性反推 arr

Java 里比较的是 jiami (arr) == result2，而 jiami (arr)=arr^ks，所以：

arr = result2 ^ ks（先用 key 生成 24 字节 keystream）。

再把 arr 逆回输入

arr 能拼回 3 个 long（小端），calculate () 的 64 轮变换是可逆的（逐轮逆推），逆回原始 3 个 64bit，再按大端拆成 24 个字符，就是正确输入。

# LOSS N

解题思路

  1. 利用 ed ≡ 1 mod φ(n)：

    1. 计算 K = e × d - 1
    2. 因为 d <φ(n)，所以 K = l × φ(n)，其中 l 是整数，且 l < e = 65537（实际上 l ≤ 65536）。

  2. 枚举 l 找到可能的 φ(n)：

    1. 遍历 l = 1 到 65536，检查是否 K % l == 0，若是则 φ_candidate = K //l。

  3. 从 φ(n) 恢复 n（利用 p 和 q 接近）：

    1. 设 s = p + q，diff = q - p
    2. 则 n = φ(n) + s - 1
    3. diff² = (s - 2)² - 4 × φ(n)
    4. 近似 s ≈ 2 + 2√φ(n)，在该值附近小范围搜索 s，使得 diff² 为完全平方且 diff > 0、(s - diff) 偶数。
    5. 计算 p = (s - diff)/2，q = (s + diff)/2
    6. 验证 p × q == n，且 q == next_prime (p)（题目特征，可确认唯一正确 n）。

  4. 恢复 flag：

    1. 用恢复的 n 计算 m = pow (c, d, n)
    2. flag = long_to_bytes(m)

这种方法不需要 n，直接从 d 和 e 恢复 φ(n) 和 n，利用 p、q 接近使搜索高效（偏移范围很小即可找到）。
