# AI 总结

以下是红日靶场 A 渗透测试的完整攻击链总结：

  1. 外网突破：
    通过 Nmap 扫描发现 Win7 开放 445 端口，利用永恒之蓝漏洞获取系统权限，关闭防火墙并创建后门用户，使用 Cobalt Strike 实现持久控制，同时发现内网网段 10.0.20.0/24。

  2. 内网横向移动：
    搭建 Socks 隧道代理穿透内网，扫描发现 Win2016 的 Redis 未授权漏洞（6379 端口），写入 Webshell 获取权限；进一步关闭其 Windows Defender 并开启 RDP，通过 Mimikatz 提取域用户凭证，发现第二层内网网段 10.0.10.0/24。

  3. 域控提权：
    利用获取的域管理员凭证（Administrator:Admin@666）和 Zerologon 漏洞（CVE-2020-1472）攻陷域控 Win2019，最终通过 SMB/RDP 完全控制域环境。

  4. 攻击链补充：
    额外验证 Win7 的通达 OA 漏洞利用和永恒之蓝攻击，以及 Redis 未授权写 Webshell 时通过 PHP 探针获取网站绝对路径的技巧。

# win2019

先做域内信息收集

[!note] NETLOGON 和 SYSVOL 都是 AD 域控制器上默认存在的共享，即使没手动创建共享，域控也会自动暴露这些资源。
重要攻击面：

  1. 这些是默认共享无法删除
  2. 管理员必须确保 SYSVOL 里不存储凭据
  3. NETLOGON 的权限设置很关键
  4. 所有用户可读，仅域管用户可写

NETLOGON ：存储用户登录脚本、组策略登录脚本以及域控制器之间进行复制的一些关键文件（如组策略模板）。当用户或计算机登录到域时，系统会根据配置从 NETLOGON 共享中检索并运行相应的脚本。所有用户可读，仅域管用户可写
​SYSVOL：存放域中所有组策略对象的核心文件。客户端（用户计算机、服务器）在应用组策略时会从这里下载必要的策略设置，通过文件复制服务在整个域内同步组策略和脚本。也存储了登录脚本的副本

安全组 权限 作用
Domain Admins 域内最高管理权限 完全控制域内所有资源（用户 / 计算机 / 组策略 / 域控制器）
Enterprise Admins 跨域最高权限（管理整个 AD 林） 控制所有子域及林级架构，可以跨域
Schema Admins 修改 AD 数据库结构 (Schema) 定义对象类型和属性
Group Policy Creator Owners 创建新的组策略对象 ，恶意 GPO 可以强制域内计算机或用户执行恶意代码
Protected Users 安全加固组，更强的认证协议 阻止弱凭证（如 NTLM）缓存
DnsUpdateProxy DNS 更新代理组

当前域只有 win2016 一个用户

域信任：不同域之间如何验证用户身份和授权访问资源，允许用户在一个域中登录，然后访问另一个域中的资源（如文件共享、应用程序等）。

# 攻击链总结

  1. 外网入口突破
    利用通达 OA 漏洞获取 Win7 权限，关防火墙开 RDP 维持访问，通过 Cobalt Strike 上线，发现内网网段（10.0.20.0/24）。

  3. 域环境控制
    凭据传递攻击（NTLM Hash: c7c654da31ce51cbeecfef99e637be15 ）或 Zerologon 漏洞攻陷域控 Win2019，提取域管明文密码（ Admin@666 ），完全掌控域环境。
